## 练习 1：设计一个 Few-shot 新闻分类器

### 目标

设计一个 Few-shot prompt，让模型将新闻标题分类为：**科技 / 体育 / 财经**。

### 要求

- 至少 3 个 shot 示例（每类一个）
- 测试 3 条新标题，验证分类准确性
- 输出格式：只回答分类名称（科技/体育/财经）

### 测试输入

```
1. "苹果发布新一代 M5 芯片，性能提升 40%"
2. "湖人队在加时赛中击败凯尔特人"
3. "央行宣布下调存款准备金率 0.5 个百分点"
```

### 提示

- 示例格式要完全一致（输入：xxx\n分类：xxx）
- 每个类别选一个典型标题作为示例
- 在 prompt 末尾留一个"现在请分类"的过渡句

### 参考答案

```python
few_shot_news = """你是一个新闻分类器。根据新闻标题判断类别，只回答分类结果。

示例 1：
输入：OpenAI 发布 GPT-5，支持实时视频理解
分类：科技

示例 2：
输入：世界杯预选赛中国队 2:1 战胜韩国队
分类：体育

示例 3：
输入：特斯拉市值突破 2 万亿美元，股价创历史新高
分类：财经

现在请分类：
输入：{news_title}
分类："""
```

**为什么这个 prompt 有效？**
- 3 个示例覆盖全部类别，格式完全一致
- 示例选的是各类别中最典型、最不容易混淆的标题
- "现在请分类"明确告诉模型：示例结束，轮到你了

## 练习 2：设计一个 CoT 推理 Prompt 解决逻辑谜题

### 目标

设计一个 Chain-of-Thought prompt，让模型解决以下逻辑谜题：

> 有 4 个人（A、B、C、D）排队。已知：
> 1. A 不在第一个
> 2. B 紧跟在 C 后面
> 3. D 在 A 前面
>
> 问：排队顺序是什么？

### 要求

- prompt 必须引导模型"一步一步思考"
- 模型需要列出每一步推理过程
- 最终给出明确的排队顺序

### 提示

- 用"请一步一步分析以下条件"开头
- 让模型逐条排除不可能的排列
- 最后要求模型给出"最终答案"

### 参考答案

```python
logic_puzzle = """请一步一步分析以下逻辑谜题，列出你的推理过程，然后给出最终答案。

谜题：有 4 个人（A、B、C、D）排队。已知：
1. A 不在第一个位置
2. B 紧跟在 C 后面（即 C 在 B 前面，且两人相邻）
3. D 在 A 前面

请按以下步骤推理：
步骤 1：根据条件 2（B 紧跟在 C），列出所有可能的 CB 组合位置
步骤 2：根据条件 1（A 不在第一个），排除不可能的排列
步骤 3：根据条件 3（D 在 A 前面），进一步缩小范围
步骤 4：综合所有条件，得出唯一解

最终答案："""
```

**为什么这个 prompt 有效？**
- 明确要求"一步一步分析"——触发 CoT 推理
- 把 3 个条件拆成 4 个推理步骤——降低模型的认知负荷
- 每个步骤只关注一个条件——避免模型同时处理多个约束导致混乱
- 最后要求"最终答案"——确保模型给出明确结论

## 练习 3（进阶）：组合技术 — Few-shot + CoT + 结构化输出

### 目标

设计一个 prompt，结合 3 种技术，让模型从用户输入中提取任务并给出优先级推理。

### 要求

- 用 Few-shot 展示一个完整的"输入 → 推理 → JSON 输出"流程
- 输出必须是合法 JSON，包含 `title`、`priority`、`reasoning` 三个字段
- `reasoning` 字段要展示 CoT 推理过程

### 参考答案

```python
advanced_prompt = """你是任务管理助手。从用户输入中提取任务信息。

示例：
输入：老板说周五之前必须把报告交上去，不然会影响季度评估。
思考过程：
1. 截止时间：周五（紧迫）
2. 后果：影响季度评估（严重）
3. 判断：紧急且重要 → priority = high
输出：{"title": "提交报告", "priority": "high", "reasoning": "截止时间紧迫（周五），且后果严重（影响季度评估），属于紧急且重要的任务。"}

现在请处理：
输入：{user_input}
思考过程："""
```

**组合效果**：
- Few-shot 让模型学会"输入 → 推理 → JSON"的完整流程
- CoT 在 `思考过程` 中展示推理
- `response_format={"type": "json_object"}` 保证输出可解析
