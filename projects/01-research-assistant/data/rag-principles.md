# RAG 检索增强生成原理

## 什么是 RAG

RAG（Retrieval-Augmented Generation）是一种将检索与生成相结合的技术。它让大语言模型能够访问外部知识库，从而提供更准确、更及时的回答。

## RAG 的核心流程

### 1. 文档分块
将长文档切分为较小的文本块（chunks），通常 200-500 字符，带重叠窗口。

### 2. 向量化
使用 Embedding 模型将文本块转换为向量表示。语义相近的文本，向量距离也相近。

### 3. 检索
用户查询向量化后，与知识库向量计算余弦相似度，取 top-k 最相关的文本块。

### 4. 增强生成
将检索到的文本块注入到 LLM 的上下文中，让模型基于这些信息生成回答。

## RAG vs 微调

| 维度 | RAG | 微调 |
|------|-----|------|
| 知识更新 | 实时更新，无需重训 | 需要重新训练 |
| 成本 | 低，只需维护向量库 | 高，需要 GPU 训练 |
| 可解释性 | 高，可追溯来源 | 低，黑盒 |
| 适用场景 | 知识密集型任务 | 特定领域风格 |

## Agentic RAG

传统 RAG 是固定管道，Agent 自主决定是否检索。当 Agent 遇到需要外部知识的问题时，才调用检索工具。这让检索更智能、更有针对性。
