# 项目4 · 智能客服 Agent（记忆 + 工具 + 人机协作）

> **综合实战**：构建一个带记忆、工具调用、情绪识别和安全防护的智能客服系统。

---

## 你会学到什么

1. **ConversationBuffer 记忆系统**：跨轮对话记忆，提取用户姓名和订单号
2. **订单查询工具**：从 `data/orders.json` 读取订单数据，按订单号查询状态
3. **情绪识别 + 转人工**：检测不满关键词（"投诉"/"退款"/"太差"等）→ 触发 Handoff
4. **防 Prompt 注入**：检测注入关键词（"忽略之前指令"/"admin"/"所有用户"等）→ 拦截
5. **离线 Mock**：API 不可用时自动降级为本地 fallback，`exit 0`

---

## 架构概览

```
用户输入
   │
   ▼
┌──────────────┐
│  安全检查    │──► Prompt 注入检测 → 拦截
└──────┬───────┘
       │
       ▼
┌──────────────┐
│  情绪检测    │──► 负面情绪关键词 → 转人工
└──────┬───────┘
       │
       ▼
┌──────────────┐   query_order(orderId)
│  记忆 + 工具 │──► 提取用户信息 → 记忆更新
│              │──► 订单查询 → 格式化结果
└──────┬───────┘
       │
       ▼
┌──────────────┐
│  LLM / Fallback │──► 生成回复
└──────────────┘
```

## 离线设计

`.env` 中 API 密钥为占位符 `sk-REPLACE-ME` 时：
- **订单查询**：纯本地 JSON 文件读取，不调 API
- **记忆系统**：纯本地数据结构，不调 API
- **情绪检测**：纯本地关键词匹配，不调 API
- **LLM 回复**：try/catch 失败 → 使用本地 fallback 回复

全程 **不依赖真实 API**，`exit 0`。

## 5 轮对话演示

| 轮次 | 用户输入 | 触发组件 |
|------|----------|----------|
| 1 | "你好，我叫张三" | 记忆：提取姓名 |
| 2 | "请帮我查一下订单 ORD-001 的状态" | 工具：订单查询 |
| 3 | "这个订单什么时候能到？" | 记忆：利用已知信息 |
| 4 | "等了这么久还没到，太差了！我要退款！" | 情绪：负面检测 + 转人工 |
| 5 | "算了，我要投诉你们的服务！" | 情绪：再次触发转人工 |
